# Day 1 – 4-Bit BCD Adder Design and Verification

## Overview

This project implements a **4-bit Binary Coded Decimal (BCD) Adder** using **Verilog HDL** and validates its functionality through simulation in **Xilinx Vivado**. The circuit performs decimal digit addition in BCD format and automatically corrects invalid results to ensure that the output remains a valid BCD representation.

## Objective

* Design a 4-bit BCD Adder using Verilog HDL.
* Implement decimal correction logic for invalid BCD sums.
* Verify the design through simulation and RTL analysis in Vivado.

## Introduction

A Binary Coded Decimal (BCD) Adder is a combinational circuit used to add decimal digits represented in BCD form. Since a single BCD digit can represent only values from 0 to 9, the direct binary sum of two BCD digits may sometimes exceed this valid range.

To maintain a valid BCD output, the circuit detects invalid results and applies a correction factor of **0110 (decimal 6)** whenever required.

## Applications

BCD Adders are commonly used in:

* Digital Clocks
* Electronic Calculators
* Digital Counters
* Financial and Accounting Systems
* Measurement Instruments
* Embedded Display Systems

## Theory

### BCD Representation

In the BCD number system, each decimal digit is represented independently using four binary bits.

| Decimal | BCD  |
| ------- | ---- |
| 0       | 0000 |
| 1       | 0001 |
| 2       | 0010 |
| 3       | 0011 |
| 4       | 0100 |
| 5       | 0101 |
| 6       | 0110 |
| 7       | 0111 |
| 8       | 1000 |
| 9       | 1001 |

Binary combinations from **1010** to **1111** do not correspond to valid BCD digits and therefore require correction.

### Operating Principle

The addition process consists of three major steps:

1. Add the two BCD digits using a 4-bit Ripple Carry Adder.
2. Determine whether the resulting binary sum is a valid BCD value.
3. If the sum exceeds decimal 9 or generates a carry, add the correction value **0110** to produce a valid BCD output.

This correction mechanism ensures accurate decimal arithmetic while maintaining BCD compatibility.

## Design Architecture

The BCD Adder was developed using a hierarchical design approach consisting of three major blocks.

### 1. First Ripple Carry Adder (RCA)

Performs the initial binary addition.

#### Inputs

* **A[3:0]**
* **B[3:0]**
* **Cin**

#### Outputs

* Intermediate Sum
* Carry Output

### 2. BCD Correction Logic

The correction circuitry determines whether an adjustment is required.

The correction condition is given by:

```text id="6ohg87"
Correction = Cout + (S3 · S2) + (S3 · S1)
```

Where:

* **Cout** = Carry generated by the first adder
* **S3, S2, S1** = Intermediate sum bits

If the correction signal becomes HIGH, the value **0110** is added to the intermediate result.

### 3. Second Ripple Carry Adder

Performs decimal correction by adding:

* Intermediate Sum
* Correction Value (0110)

#### Outputs

* Final BCD Sum
* Final Carry

## Functional Blocks

The overall design consists of:

* First Ripple Carry Adder
* BCD Correction Logic
* Second Ripple Carry Adder
* Carry Propagation Network

Together, these blocks ensure correct decimal addition and automatic correction of invalid BCD results.
